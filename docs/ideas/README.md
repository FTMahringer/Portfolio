# Portfolio Ideas Organization

This folder contains all feature ideas organized by status and priority.

## 📁 Files

### `completed.md`
Features that are already implemented and working in the portfolio. This is our "changelog" of what we've built.

### `planned.md`
Features we're actively planning or seriously considering. Organized by priority (high/medium/considering).

### `maybe.md`
Features that could be useful someday but aren't priorities. Worth keeping in mind for future consideration.

### `unnecessary.md`
Features that are cool but likely overkill for a personal portfolio. Marked as "don't implement" to avoid over-engineering.

## 🎯 Philosophy

**Keep it simple.** This is a portfolio, not an enterprise application. Features should:
- Serve an actual purpose (not just "cool tech")
- Be maintainable by one person
- Not distract from the content
- Actually get used by visitors

## 📝 Adding New Ideas

When you have a new feature idea:

1. **Ask**: Does this serve the portfolio's purpose?
2. **Check**: Is it already in one of these files?
3. **Add to `maybe.md`** if uncertain
4. **Move to `planned.md`** when you're ready to commit
5. **Move to `completed.md`** when implemented
6. **Add to `unnecessary.md`** if it's overkill

## 🚫 Red Flags (Probably Unnecessary)

Avoid features that are:
- "Enterprise-scale" tooling (Storybook, Lighthouse CI, auto-deployment bots)
- Gimmicky tech demos (Web3, AR, AI assistants)
- Over-engagement (live chat, push notifications, quizzes)
- SEO gaming (auto-submit to aggregators)
- Premature optimization (service workers, edge functions, lazy hydration)
- Nobody will use (voice navigation, haptic feedback)

## ✅ Good Portfolio Features

Look for features that:
- Showcase your work better (image galleries, case studies)
- Improve content discovery (tags, search, related posts)
- Enhance reading experience (ToC, reading progress, code highlighting)
- Are actually useful (contact form, theme switcher, comments)
- Show personality (now page, uses page, bookmarks)

---

**Remember:** A portfolio's job is to showcase your work and make you hireable. Everything else is decoration.
